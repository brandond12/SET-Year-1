Command name: wrapfile

Purpose: To wrap long lines for output to the console window

Synopsis: wrapfile [-w width] file ...

Description:

The wrapfile utility is a filter which wraps the contents of the specified files, breaking the lines to have a maximum of 40 columns (or a different maximum if the -w option is used; this maximum is called the "wrap length").

The options are as follows:

-w width Specify a line width to use instead of the default 40 columns.

Assumptions:

  • There will always be one or more filenames on the command line.
    • All filenames are guaranteed to be after all options.
  • Filenames never start with '-'.
  • The files to be taken as input for wrapping are guaranteed to be text files if they exist.
    • However, the files are not guaranteed to exist or be accessible (see Errors section).
  • The maximum length of a line found within any input file is 100 characters.
  • The files will not contain backspace characters or TAB characters or other special characters that would mess up the wrapping.

Notes:

  • If the wrap point is, for example, 40, then that means that the 41st character goes on the next line (just to clarify if the 40th character gets wrapped or not).
  • There might or might not be a space between the -w and the specified width.
  • A '\n' character is actually a carriage return character (ASCII value 0x0d) followed by a line feed character (ASCII value 0x0a).
  • Files do not necessarily end with a '\n'.
    • The reason for this is that some editing programs put the '\n' at the end and some do not.
    • If a file doesn't end with a '\n', you are required to add one (don't have the output from two different files on the same line and don't have the next command prompt on the same line).
      • I won't be testing for this.
  • If there are multiple input files on the command line, the next file should begin printing on the next line, with no blank line between files and nothing even indicating that there are multiple files (e.g. no new filename displayed and no line of asterisks).
  • If there are multiple blank lines at the end of an input file, they should all be also present in the output.
  • If width is anything other than a positive integer less than 65536 (i.e. 1 to 65535 inclusive), display a usage error message (described below).
  • You will likely get a better mark if you use Baby Steps to design this program.
    • At the very least, get the basic operation of the program working before you try to embellish it with other requirements.
  • This is NOT a group project. It must be done individually.

Errors:

  • Displays a "File not openable" error message and terminates without any other further output if any of the files cannot be opened.
    • If this happens on a second or subsequent file, it is OK to process the previous files that can be opened.
  • Displays a "Usage: wrapfile [-w width] file ..." error message and terminates if the command line options are incorrect.
    • Your error message must look exactly as stated above.
  • Displays an appropriate error message and terminates without any other further output if there is a error with reading or closing any of the files.

Exit Status:

0 successful
1 can't open file
2 command line options incorrect
4 any other fatal error
