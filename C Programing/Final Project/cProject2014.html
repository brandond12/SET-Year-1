C Programming - Project - Fall 2014

Description

Write a program that wraps lines from a text file at a certain column position.

Functional Requirements

The description of how the command should work is found separately in this dropbox at cProject_wrapfile_2014.html. 

  • In the description, anything in square brackets indicates optional items. Italicized items are required items but the words used represent something (e.g. "file" means that you need to have the name of a file). Three dots ("...") represents one or more of the preceding item (e.g. "file ..." means one or more filenames).
    • This is a common standard for the description of a command.
  • There is no guarantee that files mentioned on the command line actually exist.

Restrictions

  • Do not use global variables or goto. 
  • All code that you use must be your own. Deviating from this could result in a mark of 0.
  • Do not use exit().  Exit statuses must be handled by returning the value from main().

Bug List Requirements

  • You must create (and submit) a file called bugs.docx (or bugs.doc).
  • The top heading of the first page must state "Project Bug List for " and then state your full name.
  • This file will contain a bullet list of all bugs and missing requirements that you know of in your submitted program.
    • The list should be specific and easy to understand.
    • If you have no bugs and no missing requirements that you know of, simply state "This project is complete."
  • 10 marks of your total Project mark will be assigned to this requirement. The completeness and quality of the file will determine this mark.
    • In addition, the penalties associated with bugs or missing requirements might be lessened if they are described in this file.

File Requirements

  • Call the executable wrapfile.exe (to be renamed to wrapfile.x upon submission to the dropbox).
  • Call the file that contains your main() function wrapfile.c.
  • You can create other source files if you wish but the filenames should make the contents of the file clear.
    • All source files must have header comments. The description in all files except wrapfile.c must describe the contents of the file.
  • Put all required files in the eConestoga Project dropbox.
  • Hand in all source code and the executable.
    • Make sure that you don't mess up on this, as it'll be too late to correct it afterwards.

Late Penalties

  • Pinky bucks are not allowed.
  • You can hand in the project one day late at a 20% penalty.
  • You cannot hand the project in more than one day late.
