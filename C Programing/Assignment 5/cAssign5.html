Introduction To C
Assignment #5

Functional Requirements:

  • Write a program that creates a text file containing the printed ASCII values of the data from a binary file.  
  • Take in an input file name as the first (and only) argument on the command line using argc and argv in main(). 
    • This file must be opened as a binary file.
    • If the filename is not found on the command line, display an error message and quit the program.
  • For each character, print (to the output file) the numeric ASCII representation of the character using fprintf().
    • The output file must be called contents.txt and must be stored in the current directory. It is a text file.
    • The numeric value must be printed as an integer, three characters in size, zero-padded (use fprintf() width formatting codes for this). Put one space between each three-character integer.
  • Every 10 input characters, print a '\n' in the output file. Also put a '\n' at the end of the file.

Other Requirements

  • Define a function called getSmallFileLength() in a file you create called fileUtilities.c. It takes a name of a file as a parameter as a const char *. It must use FindFirstFile() to get the length of that file. If FindFirstFile indicates an error, display an error message and return -1. Otherwise, return the nFileSizeLow value from the WIN32_FIND_DATA struct you declare.
    • This function only works properly for files that are less than 2,147,483,647 bytes in size. This is a limitation that must be documented in both the function comment and an inline comment in the function itself.
    • The reason that you are doing this is that you have now created a utility function that you could use in other programs if you have the need.
  • Find the size of the input file using a call to getSmallFileLength() as mentioned above. The function call must have the input filename passed as a parameter.
    • If the size found is less than 1, quit the program (do not display an error message).
  • Dynamically allocate a block of memory of the appropriate size using malloc() to store the contents of the input file.
  • Read the entire file in using one fread() call. 
  • Close the files once done.
  • Free the allocated block once done.
  • Check return values for all file I/O functions and memory allocation functions as appropriate. In the case of errors, display an error message and quit the program cleanly.
  • Do not get user input from the keyboard in your program. At all. I'm serious.
  • Do not use global variables or goto.
  • Do not use exit().
  • Appropriate programming style as discussed in class must be used.
    • Header comments must be put at the top of each source file. The "Description" of the cA5.c file header must describe what the program does. The "Description" comments in the other source files must describe the contents of the file. You must include the project name in the header comment for cA5.c and cA5_proto.h

File Requirements

  • Create a project called cA5. 
    • That project should contain cA5.c, fileUtilities.c, and cA5_proto.h. 
    • It will generate cA5.exe.
  • Put all prototypes in cA5_proto.h.  Do not put any prototypes in cA5.c. 
  • Rename cA5.exe to cA5.x.
  • Put all source files and cA5.x in the appropriate dropbox in D2L.

Visual C++ issue:

  • Visual C++ will not compile your call to FindFirstFile successfully unless you make a change to the Project Properties to "Use Multi-byte Character Set" instead of using Unicode. This was covered in class. You must change this from the default in order to get your program to compile!
